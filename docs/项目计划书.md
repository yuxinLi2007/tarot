# Tarot 塔罗网页开发项目计划书

## 项目概述

- **项目名称**：韦特塔罗在线占卜网页
- **项目目标**：开发一个功能完整、用户体验良好的韦特塔罗占卜网页，支持多种牌阵、抽牌占卜和结果解读
- **核心技术**：HTML5 + CSS3 + TypeScript/JavaScript + 密码学安全随机数 + 大语言模型API
- **开发周期**：15天
- **交付成果**：可公开访问的静态网页，支持移动端和PC端访问

## 一、项目背景与技术选型

### 1.1 为什么选择韦特塔罗？

在做这个项目之前，需要先想清楚一个问题：市面上已经有很多塔罗占卜网站和App，为什么还要做一个？

**思考结论：**
- 现有产品要么功能单一（只有每日一牌），要么商业气息过重（频繁收费）
- 韦特塔罗作为最流行的塔罗体系，用户基数最大，但真正把牌面故事讲清楚的产品很少
- 技术上不复杂，但需要深度理解塔罗文化才能做好解读部分
- 可以作为一个"技术+人文"的综合项目，展示多维度能力

### 1.2 项目的核心难点与解决方案

| 难点 | 解决方案 |
|------|---------|
| 抽牌随机性保证 | 使用 CSPRNG（密码学安全随机数生成器）|
| 牌意解读个性化 | 接入 LLM API，结合问题语境生成解读 |
| 移动端体验 | 体感手势交互，无需精确点击 |
| 牌面故事呈现 | 深度挖掘韦特牌面图像叙事 |

### 1.3 设计原则

- **移动端优先**：约70%用户会通过手机访问
- **加载优先**：首屏加载时间控制在2秒以内
- **操作简单**：三步之内完成一次占卜（选牌阵→抽牌→看解读）
- **视觉沉浸**：暗黑神秘风格，减少干扰元素

### 2.2 核心技术模块

#### 2.2.1 密码学安全随机数生成器（CSPRNG）

**为什么需要CSPRNG？**

塔罗牌占卜的核心是"抽牌的随机性"。传统的伪随机数生成器（PRNG）基于确定性算法，其序列是可预测和可重现的。这在涉及概率公平性的场景中会破坏随机的公正性。

**技术方案：** 使用 `window.crypto.getRandomValues()` — 这是浏览器内置的密码学安全随机数生成器，通过汇集系统中不可预测的物理事件（如鼠标移动、键盘敲击时序、网络延迟）来生成高熵随机种子。

**为什么不让LLM直接抽牌？**

研究指出，LLM本身不擅长生成真正的随机数。如果让AI自己决定抽什么牌，它会倾向于选择"最合理"的牌而非随机牌，导致某些牌出现频率远高于正常概率。因此，正确的架构是：抽牌由CSPRNG模块独立完成，LLM只负责解读。

#### 2.2.2 塔罗牌知识图谱

每张牌需要包含以下字段：
- `id`：唯一标识（0-77）
- `name`：中文名称（如"愚者"）
- `nameEn`：英文名称（如"The Fool"）
- `type`：大阿卡纳/小阿卡纳
- `upright`：正位核心含义、关键词、详细解读
- `reversed`：逆位核心含义、关键词、详细解读
- `elemental`：元素属性（火/水/风/土）
- `astrological`：对应星座
